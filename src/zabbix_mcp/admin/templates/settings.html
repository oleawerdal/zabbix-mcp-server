⚠️ All changes in this section require a server restart to take effect.

IP address for MCP and admin portal. 127.0.0.1 = localhost only, 0.0.0.0 = all interfaces.
MCP Listen Port 🛈
TCP port for the MCP protocol endpoint (/mcp).
Transport 🛈
MCP transport protocol. HTTP is recommended for network clients.
Log Level 🛈
{% set current_log_level = settings.log_level | default('info') | lower %}
Log File 🛈
Path to log file. Empty = stderr only. Note: changing this path also requires updating logrotate config manually.
Compact Output 🛈
Response Max Chars 🛈
Max characters per tool response before truncation (default: 50000, min: 5000). Increase for template export workflows (e.g. 200000 for medium templates, 500000 for large built-in templates).
Public URL 🛈
External URL clients use to reach this MCP server. Required when host is 0.0.0.0 and the server is exposed via a public DNS name or reverse proxy. Used in OAuth discovery (advertised to MCP clients during auth) and as the base URL in the Client MCP Wizard. Leave empty to auto-derive from {scheme}://{host}:{port}. No trailing path - the /mcp or /sse suffix is appended automatically. Must be https:// when TLS is enabled.

⚙️ Admin Portal

⚠️ All changes in this section require a server restart to take effect.
Admin Portal 🛈
The portal is on - that's how you got here. Toggling it off from this UI is intentionally blocked because it would lock you out. To disable: SSH to the server, set [admin].enabled = false in /etc/zabbix-mcp/config.toml, then sudo systemctl restart zabbix-mcp-server.
Admin Port 🛈
Separate port for admin UI. Shares host and TLS with MCP server.
Check for updates 🛈
When on, this server pings the GitHub releases API at boot, after every successful admin login (throttled to once per 60 sec), and on demand via the “Check now” button next to the version pill. Shows an “Update vX.Y available” banner in the top bar when a newer release lands. Outbound HTTPS to api.github.com; no other data leaves the server. Off-line / restricted networks should disable this.

✨ AI Template Generation

Server-side default for the “Generate with AI” button on the report template editor. Set a provider and key here so all admins share one configuration. If you leave the provider on “None”, each admin has to paste their own key in the AI modal every time.
Enabled
When off, the “Generate with AI” button is hidden from the template editor even if a key is configured.
Provider 🛈
Picking a provider preselects its default model and endpoint. Ollama allows empty API key for local unauthenticated setups.
API Base URL 🛈
Required for Azure OpenAI (deployment URL) and self-hosted Ollama. Locked for providers with a canonical endpoint.
API Key 🛈
{% if settings.ai_api_key_configured %} A key is already stored. Leave this field blank to keep it, or paste a new one to rotate. {% else %} Paste your Anthropic or OpenAI API key. Stored in [admin.ai].api_key with file permissions 0600. {% endif %}
Model 🛈
Leave empty for the provider's default. Provider defaults: Anthropic claude-sonnet-4-6, OpenAI gpt-5, Gemini gemini-2.0-flash, Ollama llama3.2, Mistral mistral-small-latest, Groq llama-3.3-70b-versatile.
Timeout (seconds) 🛈
How long to wait for the LLM to respond before giving up. Default 180 s. Large reasoning models (Claude Opus, GPT-5) regularly need 90-150 s for a full template; bump to 300 s if you keep hitting "read operation timed out".
Max tokens 🛈
Upper bound on the template length the LLM may produce. Default 8000 is enough for a full A4 report; raise only if generated output gets cut off.
